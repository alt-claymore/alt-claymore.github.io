Что означает парамтер Esm

Параметр -esm - позволяет указать режим Стратум протокла для подключения к пулу.

Значения esm

  1. -esm 0 - режим eth-proxy (по умолчанию) для добычи eth,
  2. -esm 1 - qtminer mode (например, ethpool.org), (режим qtminer используется для включения режима, в котором будет отображаться дополнительная информация об шарах (принято / отклонено),
  3. -esm 2 — режим miner-proxy (например, coinotron.com),
  4. -esm 3 — режим для nicehash.

Параметр -esm поможет устранить проблемы с подключением майнера к пулу.

Примеры использования

Режим eth-proxy (по умолчанию), значение можно не указывать:

EthDcrMiner64.exe -epool eth-eu2.nanopool.org:9999 -ewal 0x1a0e2c4cd699cee12672adc223fdb30b93253eba -eworker claymore -epsw x -mode 1 -r 0 -dbg -1 -mport 0 -etha 0 -retrydelay 1 -tt 79 -ttli 85 -tstop 89 -fanmin 30
pause

Режим ESM 1 (подойдет для пула ethermine):

EthDcrMiner64.exe -epool eu1.ethermine.org:4444 -ewal 0x9147460980c93629e775783148591b7d0a0cbf2d -eworker asus -epsw x -mode 1 -dbg -1 -mport 0 -etha 0 -retrydelay 1 -tt 75 -ttli 85 -tstop 89 -esm 1
pause

Режим ESM 2 (подойдет для пула miningpoolhub):

EthDcrMiner64.exe -epool europe.ethash-hub.miningpoolhub.com:20535 -ewal rgzrgz_leo.rgzrgz_leo -epsw x -mode 1 -dbg -1 -mport 0 -etha 0 -retrydelay 1 -tt 75 -ttli 85 -tstop 89 -esm 2
pause

Режим ESM 3 (подойдет для пула nicehash):

EthDcrMiner64.exe -epool stratum+tcp://daggerhashimoto.eu.nicehash.com:3353 -ewal 3HHG3zpxP4AmbKDR9iRy1c2bqAb4WKhNwm -eworker claymore -epsw x -mode 1 -r 0 -dbg -1 -mport 0 -etha 0 -retrydelay 1 -tt 79 -ttli 85 -tstop 89 -fanmin 30 -allcoins 1 -esm 3
pause

Как это работает внутри программы?

Большая часть программного обеспечения для майнинга (включая Claymore) предназначена для подключения к прокси-серверу, потому что именно этим пользуются пулы, так как было бы небезопасно открывать свои узлы в Интернете.

 Эти прокси-серверы реализуют такие вещи, как аутентификация (имена пользователей / воркеров, пароли и т. д.), Если пул требует определенный протокол и он не реализован, то пул будет принимать соединения, пытающиеся использовать его функции.
